# RedNote Translate 设计文档

## 概述

像素级复刻 Kagi Translate 的"LinkedIn Speak"功能，但针对小红书风格。用户输入普通文字，输出小红书体文案。

## 产品定位

- **目标用户**：需要快速生成小红书风格文案的内容创作者、运营人员、或纯粹觉得好玩的人
- **核心价值**：一键把"人话"翻译成"小红书话"
- **差异化**：市面上都是"文案生成器"，没有简洁的"翻译器"界面

## 功能设计

### 核心功能

1. **文本翻译**
   - 输入：普通文字（中文/英文/书面语）
   - 输出：小红书风格文案

2. **多种输出风格**
   | 风格 | 特点 | 示例场景 |
   |------|------|----------|
   | 标准小红书体 | emoji爆炸 + 绝绝子 + 姐妹们 | 通用 |
   | 种草体 | 强调产品好用、必买 | 产品推荐 |
   | 探店体 | 美食描述、环境氛围 | 餐厅/咖啡店 |
   | 情感体 | 走心、治愈、扎心 | 情感内容 |
   | 干货体 | 分点、总结、实用 | 教程/攻略 |
   | 炫富体 | 低调炫耀、凡尔赛 | 生活分享 |

3. **一键复制**
   - 复制纯文本
   - 复制带格式（保留emoji）

## 技术架构

### 技术栈

- **前端**：Next.js 14+ (App Router)
- **样式**：Tailwind CSS
- **部署**：Vercel
- **LLM**：DeepSeek API（兼容 OpenAI 格式）

### 架构图

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Browser   │────▶│  Next.js    │────▶│  DeepSeek   │
│  (Frontend) │◀────│  API Route  │◀────│    API      │
└─────────────┘     └─────────────┘     └─────────────┘
```

### API 设计

**POST /api/translate**

请求：
```json
{
  "text": "今天吃了个汉堡",
  "inputType": "normal",
  "outputStyle": "standard"
}
```

响应：
```json
{
  "result": "姐妹们！！！这个汉堡真的绝绝子...",
  "style": "standard"
}
```

### Prompt 工程

每种风格对应一个 system prompt，定义：
- 风格特点
- 常用词汇/表达
- emoji 使用规则
- 示例输出

示例（标准小红书体）：
```
你是一个小红书文案翻译器。把用户输入的普通文字翻译成典型的小红书风格。

特点：
- 大量使用emoji（每句话至少1-2个）
- 夸张表达：绝绝子、yyds、真的会谢、DNA动了、谁懂啊
- 称呼：姐妹们、宝子们、家人们
- 感叹号多用！！！
- 适当使用换行增加可读性
- 结尾加 #标签

示例：
输入：今天吃了个汉堡
输出：姐妹们！！！这个汉堡真的绝绝子！🍔✨ 一口下去直接DNA动了！！肉汁在嘴里爆开的感觉谁懂😭😭 不允许还有人没吃过！真的会谢！！#美食探店 #今日份快乐
```
